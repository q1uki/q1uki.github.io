Requirements

The task requires designing a network architecture with security features for a medium-sized enterprise with 200 employees. The company has three departments: Sales, Development, and Management. It needs to provide internal remote access, and a DMZ to host web, email, and DNS servers. The network architecture should use a three-layer structure: core, distribution, and access, with two or more Layer 3 switches. Appropriate VLANs and IP ranges need to be designed. Internal/external firewalls, routers, and switches should be configured for security and access control. Access points include employee devices and guest APs. Virtual machine network configuration will simulate a cloud environment, and Kali will be used to simulate a public network attacker for security testing.

Network Design

Actually, using a redundant design would be better to meet the requirements, but due to time constraints, this design only implements a structure where two Layer 3 switches serve as the core and distribution layers, respectively. While a single Layer 3 switch can perform both core and distribution functions, separating them can improve processing efficiency and better meet real-world demands. The core layer focuses on high-speed, reliable data forwarding and is deployed in a central equipment room (e.g., a school network center). The distribution layer is responsible for aggregating traffic from multiple Layer 2 switches (access layer), routing VLANs, and enabling ACLs (Access Control Lists) to control VLAN access permissions, deployed in a departmental equipment room (e.g., a dormitory building’s main equipment room).

Layer 3 Analysis Reference: https://libai.blog.csdn.net/article/details/151764710

In terms of security design, a three-legged firewall architecture is used here, where the DMZ switch is directly connected to the firewall, ensuring that communication between the internal network and the DMZ is protected by the firewall. However, this may degrade the performance of internal hosts accessing DMZ services, as all access must pass through the firewall. There is also a dual-firewall solution, where an external firewall filters traffic entering from the public network, and an internal firewall protects communication between the DMZ and the internal network.

Firewall Architecture Design Reference: https://www.firewall.cx/networking/network-fundamentals/firewall-topologies-dmz-zone.html

DMZ, Demilitarized Zone, is an isolated area used to deploy servers accessible from both internal and external networks. Because the deployed services face untrusted external traffic, they are also “contaminated” and become untrusted, while simultaneously needing to provide services to the internal network. To securely isolate internal and external network traffic, and prevent external traffic from directly penetrating servers to reach the internal network, a DMZ is necessary. Even if a server in the DMZ is compromised, it cannot directly move laterally into the internal network.

Peculiar Errors

After GNS3 has been running for a long time, situations may arise where the configuration is correct but connectivity fails. You can right-click on the connection line to capture packets for inspection. Sometimes, network cables (the lines connecting two devices) appear connected but are actually disconnected, or device network ports act up, etc. When in doubt, restart GNS3; it saves a lot of time. (Spent half a day troubleshooting DHCP only for a restart to fix it, I was furious)

Improvements

  1. There is only one distribution and aggregation switch, meaning all traffic is forwarded through a single Layer 3 switch. This limits performance and introduces a single point of failure (if one fails, everything goes down). Given enough time, a redundant structure should be implemented. In a truly redundant architecture, the distribution layer typically has two Layer 3 switches. By configuring HSRP (Hot Standby Router Protocol) or VRRP (Virtual Router Redundancy Protocol), a unified virtual gateway IP (e.g., 10.0.10.1) is created. This way, if one distribution switch fails, the other instantly takes over the gateway, with users experiencing no interruption.
  2. Although some security settings have been implemented, they are very basic and only meet the needs for learning and experimentation. In real-world applications, advanced detection and defense systems such as IDS should be employed.
