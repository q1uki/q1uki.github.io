【Study Notes】Communication Characteristics of Six RAT (Remote Access Trojan) Protocols

1 Gh0st

1.1 Basic Information

Gh0stRAT was initially developed by the C.Rufus team. The client runs as a service, can auto-start with the system, and evades antivirus detection by hiding files and processes. It primarily includes modules for file management, system information collection, process management, screen capture, and remote control. Additionally, it has the ability to steal passwords, capture network traffic, and execute remote CMD commands. It is typically spread through email attachments, malicious websites, or software vulnerabilities.

The trojan was first discovered in 2009. Its 3.6 version source code was made freely available online, making it popular and leading to many modified variants, such as Damauli (Big Bad Wolf). In 2018, the Iron Tiger organization was observed using a Gh0stRAT variant in attack campaigns, and in 2021, a Gh0stRAT variant was observed planting backdoors on user computers via fake Telegram installers.

1.2 Basic Characteristics

  1. TCP transmission, default listening port 80, commonly 8000, 443, disguised as Microsoft Device Manager service.
  2. Data (payload) header displays a magic number in plaintext. E.g., Gh0st, GOLDt, HEART, Level, BeiJi.
  3. A fixed magic number exposes its version information.
  4. It also responds to some illegal requests outside of protocol specifications, such as GET / HTTP/1.0\r\n\r\n.
  5. Lacks effective packet validation methods.
  6. No reasonable session management mechanism.
  7. In the source code, the m_iocpServer() method is responsible for communication packet transmission. Although Gh0stRAT has multiple variants, this method is a core part of each kernel and is usually not modified, so all variants share a unified communication characteristic.

1.3 Communication Characteristics

Environment setup reference: gh0st3.6 + vc6.0 + windows xp compilation process

Throughout the communication process, the TCP flags of all traffic packets used for data transmission include PSH. In the Gh0stRAT handshake, the controlled client first initiates a TCP connection request. After a three-way handshake establishes the TCP connection, the controlled client sends a payload prefixed with “Gh0st\xae\x00\x00\x00\xe0\x00\x00\x00\x78\x9c“ and containing encoded and obfuscated system information for going online.

Upon receiving the online information, the main controller responds with a traffic packet indicating confirmation, with the payload also prefixed with “Gh0st \xae\x00\x00\x00\xe0\x00\x00\x00\x78\x9c“. The 14th-15th bytes “\x78\x9c“ are the zlib compression header, located at a fixed position in Gh0st communication packets, see Gh0st Damauli RAT Family Communication Protocol Analysis.

2 DarkcometRAT

2.1 Basic Information

DarkcometRAT, also known as “Dark Comet,” was developed by Jean-Pierre Lesueur in 2008. In addition to common encryption and obfuscation techniques found in remote access trojans, it also includes rootkit functionality, allowing it to hide its presence in the system. In 2012, the trojan’s author ceased updating it, with the latest version being 5.4.1.

Currently active trojans predominantly use dynamic domain names as server domains, with “ddns.net” and “duckdns.org” accounting for over 50%. In a recent monthly threat intelligence report by Tencent Anti-Virus Lab, Darkcomet was found to be the most used malicious network protocol that month, with a single-day peak of 3,000 instances.

2.2 Basic Characteristics

  1. TCP transmission, default listening port 1604.
  2. RC4 algorithm for encrypting and decrypting communication traffic; main controller and controlled client must use the same key to receive and send information.
  3. Uses predefined command words for communication, with parameters separated by “|”.
  5. The TCP flags of all traffic packets involved in data transmission include PSH.

2.3 Communication Characteristics

After completing the TCP three-way handshake, the main controller sends the RC4-encrypted command word “IDTYPE“ using a predefined key.

Default keys are version-dependent and generally in the format “#KCMDDC𝑥#-8900123456789“, where 𝑥 is the version number, and “0123456789“ is a user-modifiable field.

Subsequently, the controlled client responds with the ciphertext of the command word “SERVER“. After the main controller receives it, it responds with the command word “GetSIN“ to request the controlled client’s host information and the IP address of the database receiving the information. The controlled client then responds with its own information, completing the Darkcomet handshake.

2.4 Platform Comparison

3 njRAT

3.1 Basic Information

njRAT, also known as Bladabindi, was first discovered in 2013 and developed using the .NET framework.
In October 2016, Softpedia reported that a cracked version of VMware downloaded and installed njRAT via Pastebin, and terminating the process would cause the computer to crash.
In January 2023, a trojan virus spread in the Middle East, with attackers using .cab files for “political dialogue,” which, when opened, would launch a .vbs script to download and install njRAT locally from the cloud.

3.2 Basic Characteristics

  1. TCP transmission, default listening port 1177 for version 0.6.4, different versions have different default ports.
  2. The TCP flags of all traffic packets involved in data transmission include PSH.
  3. Parameters are separated by '|'|'|'.

3.3 Communication Characteristics

① Client -> Server: Go online
The controlled client actively initiates a TCP connection to the main controller. After the TCP three-way handshake is completed, it sends its own information to go online. Parameters are separated by “|'|'|“, and the suffix keyword is “[endof]“.

4 GreameRAT

4.1 Basic Information

GreameRAT was released in 2012, originated from Morocco, and developed using Pascal with Delphi’s Unicode version. It also features a modular architecture, allowing attackers to add or remove functionalities as needed.

4.2 Basic Characteristics

GreameRAT uses TCP transmission. Except for version 1.9, all other versions have default listening ports of 777, 888, 999; version 1.9 only has 999. In the handshake communication between the controlled client and the main controller, traffic packets involving data exchange all include the PSH TCP flag, with the default key being “123456”.

4.3 Communication Characteristics

In the GreameRAT handshake, the controlled client first sends a number containing the key length ‘n’. In the testing environment, 25 was found to be a fixed constant, so 25+n is the number sent, which was 31 in this handshake, with a suffix of “\x7c\0a“.

The controlled client immediately follows by sending key-related verification data.

Only when both pieces of data are correct will the main controller respond sequentially with “\x20\x0d\x0a“ to indicate receipt, followed by data with the suffix “###@@@“, completing the handshake.

Additionally, it was found that if the controlled client sends other data besides the correct verification data, the main controller will respond with “tentarnovamente|“, which is Portuguese for “try again.”

5 QuasarRAT

5.1 Basic Information

QuasarRAT was developed by GitHub user MaxXor using C#, first publicly released in July 2014, and was formerly known as “xRAT”. It primarily targets Windows operating systems and maintains persistence through scheduled tasks and registry entries.
In February 2018, researchers observed an attack campaign that distributed QuasarRAT as the final payload via malicious RTF files; in May 2019, enSilo observed the APT10 group using PlugX and QuasarRAT to attack government and private organizations in Southeast Asia. Furthermore, the well-known APT group “Mahakala” (摩诃草) has also long used this tool.

5.2 Basic Characteristics

QuasarRAT uses TCP transmission, with a common listening port of 5678. In traffic packets related to data transmission, the TCP flags all include PSH.

6 XtemeRAT

6.1 Basic Information

XtremeRAT was released in 2010, developed using Delphi’s Unicode version, and open-sourced on GitHub. Its main supported functions include file and process management, keylogging, service management, and registry management. It was once used by the North African Fox (APT-C-44) group in attacks targeting Arabic-speaking regions.

6.2 Basic Characteristics

XtremeRAT uses TCP transmission, with default listening ports of 80, 81, 82. The default password is 1234567890.

6.3 Communication Characteristics

The controlled client actively initiates a TCP handshake. After completing the TCP three-way handshake to establish a connection, the controlled client first sends version information “myversion|3.2“.

The main controller replies with the first response “X\n\r“ as a response prefix, which is a communication characteristic of XtremeRAT.

Next, the main controller replies with the second response. In the payload, bytes 1-8 are the key (password is 1234 in this example), bytes 9-16 are the zlib compressed file size (message length is 152 bytes in this example), both represented in little-endian. Bytes 17-18 are the zlib indicator “\x78\x01“, which is fixed in XtremeRAT communication.
